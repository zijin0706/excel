## 一、工具概述

### 这个工具做什么？

把多个 Excel/CSV 文件按照你指定的规则进行匹配比对，自动输出：

- **匹配成功的** → `data/output/matched/` 文件夹
- **匹配失败的** → `data/output/unmatched/` 文件夹

### 支持多大的数据量？

- 千万级数据（1000 万行）正常处理
- 输出自动拆分，每个文件不超过 30 万行，确保 Excel 能打开

### 整体流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  准备数据     │ ──→ │  Excel→CSV   │ ──→ │  编写规则     │ ──→ │  运行脚本     │
│  放到 input/  │     │ 双击 convert │     │ config.yaml  │     │ 双击 run     │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
                                                                        │
                                                                        ▼
┌──────────────┐     ┌──────────────┐
│ 输出 Excel    │ ←── │  自动拆分     │
│ matched/      │     │ 每文件≤30万行 │
│ unmatched/    │     │              │
└──────────────┘     └──────────────┘
```

### 文件夹结构

```
excelUtils/                   ← 工具主目录
├── setup.sh / setup.bat      ← 首次安装用（只需一次）
├── convert.sh / convert.bat  ← Excel 转 CSV（双击）
├── run.sh / run.bat          ← 运行匹配（双击）
├── config.yaml               ← 【你需要编辑的】匹配规则
├── data/
│   ├── input/                ← 【把 Excel/CSV 放这里】
│   └── output/
│       ├── matched/          ← 匹配成功的输出
│       └── unmatched/        ← 匹配失败的输出
```

## 二、环境安装（只需一次）

### Mac 系统

**第 1 步：安装 Python**

打开 https://www.python.org/downloads/ ，下载 macOS 版本，双击安装。

安装完成后打开「终端」（在启动台搜索"终端"），输入：
```bash
python3 --version
```
看到 `Python 3.x.x` 就表示安装成功。

**第 2 步：安装工具依赖**

在终端中，进入工具目录：
```bash
cd 此处替换为工具文件夹路径/excelUtils
```

运行安装脚本：
```bash
bash setup.sh
```

看到「安装成功」即可。

---

### Windows 系统

**第 1 步：安装 Python**

1. 打开 https://www.python.org/downloads/ ，点击黄色按钮下载
2. 双击下载的文件，**务必勾选底部的「Add Python to PATH」**，然后点 Install
3. 安装完成后，按 `Win + R`，输入 `cmd` 回车，输入：
```
python --version
```
看到 `Python 3.x.x` 就表示安装成功。

**第 2 步：安装工具依赖**

在文件资源管理器中打开工具文件夹，在地址栏输入 `cmd` 回车，然后输入：
```
setup.bat
```

看到「安装成功」即可。

## 三、Excel 转 CSV

> 如果你的数据已经是 `.csv` 格式，跳过这一步，直接看第四章。
>
> 如果你的数据是 `.xlsx` / `.xls` 格式，**强烈建议先转成 CSV**。千万级数据的 xlsx 读取非常慢且容易出错，转成 CSV 后速度快几十倍。

### 一键转换

把 `.xlsx` 文件放到 `data/input/` 后：

- **Mac**：双击 `convert.sh`
- **Windows**：双击 `convert.bat`

脚本会自动扫描 `data/input/` 下所有 `.xlsx` 和 `.xls` 文件，转成同名的 `.csv` 文件。

### 转换效果

```
转换前                              转换后
data/input/                        data/input/
├── 客户表.xlsx                     ├── 客户表.xlsx    ← 原文件保留
├── 订单表.xlsx          →          ├── 客户表.csv     ← 新生成的
└── 销售记录.xlsx                    ├── 订单表.xlsx
                                    ├── 订单表.csv
                                    ├── 销售记录.xlsx
                                    └── 销售记录.csv
```

### 高级用法（仅 Mac 终端）

```bash
# 指定输入输出目录
python3 -m excel_utils.xlsx2csv ./xlsx文件目录 ./csv输出目录

# 只转换指定 Sheet
python3 -m excel_utils.xlsx2csv ./data/input ./data/input -s Sheet2

# 递归搜索子目录
python3 -m excel_utils.xlsx2csv ./data/input ./data/input -r
```

## 四、快速上手（5 分钟）

### 示例：两个文件做匹配

**场景**：你有 `客户表.csv` 和 `订单表.csv`，要找出哪些客户有订单、哪些没有。

**第 1 步**：把文件放到 `data/input/` 里

```
data/input/
├── 客户表.csv
└── 订单表.csv
```

**第 2 步**：编辑 `config.yaml`

打开 `config.yaml`（用记事本或任何文本编辑器），修改为：

```yaml
version: 1

settings:
  input_dir: "./data/input"
  output_dir: "./data/output"
  chunk_size: 300000
  encoding: "utf-8"
  duckdb:
    memory_limit: "8GB"
    threads: 4

# 你的数据文件
sources:
  - name: customers
    files: "客户表.csv"
    format: csv

  - name: orders
    files: "订单表.csv"
    format: csv

# 匹配规则
pipeline:
  - step: c
    from: customers

  - step: o
    from: orders

  # 匹配成功的
  - step: matched
    join:
      type: inner
      left: c
      right: o
      keys:
        - [客户ID, 客户ID]    # ← 改成你实际的列名

  # 客户无订单的（匹配失败）
  - step: cust_no_order
    join:
      type: anti_left
      left: c
      right: o
      keys:
        - [客户ID, 客户ID]

  # 订单无客户的（匹配失败）
  - step: order_no_cust
    join:
      type: anti_right
      left: c
      right: o
      keys:
        - [客户ID, 客户ID]

# 输出
outputs:
  - name: 匹配成功
    from: matched
    path: "matched/"
    prefix: "匹配成功"
    format: xlsx

  - name: 客户无订单
    from: cust_no_order
    path: "unmatched/"
    prefix: "客户无订单"
    format: xlsx

  - name: 订单无客户
    from: order_no_cust
    path: "unmatched/"
    prefix: "订单无客户"
    format: xlsx
```

**第 3 步**：运行

- **Mac**：双击 `run.sh`
- **Windows**：双击 `run.bat`

结果在 `data/output/` 里。

## 五、配置文件编写指南

配置文件 `config.yaml` 分为四个部分：

### 5.1 settings（全局设置）

```yaml
settings:
  input_dir: "./data/input"     # 数据文件放哪里
  output_dir: "./data/output"   # 结果输出到哪里
  chunk_size: 300000            # 每个输出文件最多多少行
  encoding: "utf-8"             # 文件编码，中文系统用 utf-8
  duckdb:
    memory_limit: "8GB"         # 内存限制，数据量超大时可调大
    threads: 4                  # 并行数，CPU 核心数以内
```

### 5.2 sources（数据源）

```yaml
sources:
  # CSV 文件
  - name: 客户表                   # 取个名字，后面引用用
    files: "客户*.csv"             # 文件名，支持 * 通配符
    format: csv

  # Excel 文件
  - name: 订单表
    files: "订单数据.xlsx"
    format: xlsx
    sheet: "Sheet1"               # Excel 页签名称

  # 多个文件（同名模式）
  - name: 销售记录
    files: "sales_*.csv"          # 匹配 sales_2024.csv、sales_2025.csv 等
    format: csv
```

**文件名通配符说明：**

| 写法 | 匹配 |
|------|------|
| `客户表.csv` | 精确匹配这个文件 |
| `客户*.csv` | 匹配 客户A.csv、客户数据.csv 等 |
| `data/*.csv` | 匹配 data 子目录下的所有 csv |
| `**/*.xlsx` | 匹配所有子目录下的 xlsx |

### 5.3 pipeline（处理步骤，按顺序执行）

支持三种步骤类型：

#### 类型 1：加载 + 过滤（from + filter）

```yaml
  - step: 步骤名
    from: 数据源名称
    filter: "金额 > 100 AND 状态 = '已确认'"   # 可选
```

#### 类型 2：表关联匹配（join）

```yaml
  - step: 步骤名
    join:
      type: inner                    # 匹配类型，见下文
      left: 左表步骤名
      right: 右表步骤名
      keys:                          # 用哪些列匹配
        - [左表列名, 右表列名]         # 单列
        # - [列A, 列B]                # 可多列
      conditions:                    # 可选的额外条件
        - "L.金额 >= R.最低金额"
      filter: "金额 > 1000"          # 匹配后再过滤
```

#### 类型 3：自定义 SQL（高级用法）

```yaml
  - step: 步骤名
    sql: |
      SELECT
        主数据编码,
        COUNT(*) AS 实际数量
      FROM 上一步骤名
      GROUP BY 主数据编码
```

### 5.4 outputs（输出设置）

```yaml
outputs:
  - name: 输出名称
    from: 引用上一步的步骤名
    path: "matched/"              # 子目录
    prefix: "匹配成功"             # 文件名前缀
    format: xlsx                  # xlsx 或 csv
```

## 六、匹配模式详解

### 6.1 五种匹配类型

假设左表是客户表，右表是订单表，匹配列为「客户ID」：

| type | 含义 | 输出什么 | 典型场景 |
|------|------|---------|---------|
| `inner` | 两边都有的 | 有订单的客户 | 找出匹配成功的记录 |
| `left` | 左表全部 + 右表匹配 | 所有客户，没订单的订单列为空 | 以左表为准补全信息 |
| `right` | 右表全部 + 左表匹配 | 所有订单，没客户的客户列为空 | 以右表为准补全信息 |
| `anti_left` | 左表有、右表没有 | 没有订单的客户 | 找出左表缺失项 |
| `anti_right` | 右表有、左表没有 | 找不到客户的订单 | 找出右表缺失项 |

### 6.2 示意图

```
客户表:  [C001] [C002] [C003] [C004]
订单表:  [C001] [C002]          [C005]

inner:       C001, C002               → 2 条，两表都有的
anti_left:             C003, C004     → 2 条，客户表有但订单表没有
anti_right:                  C005     → 1 条，订单表有但客户表没有
```

### 6.3 多条件匹配

```yaml
join:
  type: inner
  left: a
  right: b
  keys:
    - [客户ID, 客户ID]
    - [日期, 日期]              # 两列同时匹配
  conditions:
    - "L.金额 > R.最低金额"     # 额外的 SQL 条件
```

### 6.4 分组聚合匹配（高级场景）

当不是简单的行对行匹配，而是需要先分组统计再比对时：

```yaml
pipeline:
  - step: raw
    from: 数据源

  # 先分组统计
  - step: stats
    sql: |
      SELECT
        split_part(编码, '_', 1) AS 主编码,
        count(*) AS 实际数量,
        max(应有数量) AS 应有数量,
        sum(金额) AS 总金额
      FROM raw
      GROUP BY split_part(编码, '_', 1)

  # 再拆成成功/失败
  - step: success
    from: stats
    filter: "实际数量 = 应有数量"

  - step: fail
    from: stats
    filter: "实际数量 != 应有数量"
```

## 七、输出说明

### 文件位置

```
data/output/
├── matched/           ← 匹配成功的
│   ├── xxx_001.xlsx
│   ├── xxx_002.xlsx   ← 超过 30 万行自动拆分
│   └── ...
└── unmatched/         ← 匹配失败的
    └── ...
```

### 文件命名规则

`{prefix}_{序号}.xlsx`

例如 `prefix: "匹配成功"` → `匹配成功_001.xlsx`、`匹配成功_002.xlsx`

### 列名重复处理

如果两表有同名列（如都有"客户ID"），JOIN 后右表的列会自动加 `_1` 后缀：

| 输出列名 | 来源 |
|---------|------|
| 客户ID | 左表 |
| 客户ID_1 | 右表 |
| 姓名 | 左表 |
| 金额 | 右表 |

如需避免，可在加载步骤中用 `select` 重命名（见高级用法）。

## 八、常见问题

### Q1：双击 run 闪退怎么办？

右键点击 `run.bat`/`run.sh`，选择「在终端中打开」或「编辑」，查看错误信息。

### Q2："No files matched for source"

意思是 `files` 配置的通配符没有匹配到任何文件。检查：
- 文件名是否写对了
- 文件是否放到了 `data/input/` 里
- 通配符是否正确（如 `*.csv` 不是 `*.CSV`）

### Q3：输出 Excel 打不开

可能是文件太大。检查 `chunk_size` 是否设置了（默认 300000）。

### Q4：处理很慢怎么办？

1. 把 `.xlsx` 转成 `.csv` 再处理（xlsx 慢很多）：
   ```
   python3 -m excel_utils.xlsx2csv ./data/input ./data/input
   ```
2. 增加 `threads`（改成 CPU 核心数）
3. 增加 `memory_limit`

### Q5：列名包含中文/特殊字符怎么办？

DuckDB 支持中文列名。在 `filter` 和 `sql` 中直接用中文列名即可，不需要引号。

### Q6：多个配置文件如何切换？

```bash
# Mac
bash run.sh config_场景A.yaml
bash run.sh config_场景B.yaml

# Windows：把配置文件拖到 run.bat 上
```

## 附录：完整配置模板

```yaml
version: 1
settings:
  input_dir: "./data/input"
  output_dir: "./data/output"
  chunk_size: 300000
  encoding: "utf-8"
  duckdb:
    memory_limit: "8GB"
    threads: 4

sources:
  - name: table1
    files: "文件1.csv"
    format: csv
  - name: table2
    files: "文件2.csv"
    format: csv

pipeline:
  - step: a
    from: table1
    filter: ""
  - step: b
    from: table2
    filter: ""
  - step: matched
    join:
      type: inner
      left: a
      right: b
      keys:
        - [关键列, 关键列]
  - step: unmatched_a
    join:
      type: anti_left
      left: a
      right: b
      keys:
        - [关键列, 关键列]
  - step: unmatched_b
    join:
      type: anti_right
      left: a
      right: b
      keys:
        - [关键列, 关键列]

outputs:
  - name: 匹配成功
    from: matched
    path: "matched/"
    prefix: "匹配成功"
    format: xlsx
  - name: 表1未匹配
    from: unmatched_a
    path: "unmatched/"
    prefix: "表1未匹配"
    format: xlsx
  - name: 表2未匹配
    from: unmatched_b
    path: "unmatched/"
    prefix: "表2未匹配"
    format: xlsx
```
